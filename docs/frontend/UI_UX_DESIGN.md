### 1.3 使用情境 (User Scenarios)

#### 情境 1：牧師準備講道

> 馬牧師正在準備下週日關於「饒恕」的講道。他打開 Bible RAG，在問答框輸入「聖經怎麼談饒恕？」系統回傳 LLM 整理的回答，並列出相關經文段落如馬太福音 18:21-35「饒恕七十個七次」。他點擊段落查看完整經文，並透過知識圖譜探索「彼得」相關的其他教導。

**使用者旅程**:
1. 進入首頁 (問答頁面)
2. 輸入問題「聖經怎麼談饒恕？」
3. 等待 5-8 秒，查看 LLM 回答
4. 瀏覽相關經文段落列表
5. 點擊「饒恕七十個七次」段落
6. 閱讀段落詳情頁的完整經文
7. 點擊「彼得」進入知識圖譜
8. 探索彼得相關的其他經文和教導

---

#### 情境 2：神學生研究人物

> 以恩正在撰寫關於摩西的報告。她進入知識圖譜頁面，搜尋「摩西」後查看摩西的詳細資訊和相關人物關係。她透過視覺化圖譜發現摩西與亞倫、米利暗、法老的關係，並點擊各連結深入閱讀相關經文。

**使用者旅程**:
1. 從導航進入「圖譜」頁面
2. 在圖譜搜尋框輸入「摩西」
3. 點擊搜尋結果選擇「摩西」
4. 查看摩西的實體詳情面板
5. 觀察力導向圖呈現的關係網絡
6. 點擊「亞倫」節點查看詳情
7. 點擊相關經文連結閱讀內容

---

#### 情境 3：信徒日常靈修

> 美玲每天早上使用手機進行靈修。她打開 Bible RAG，進入瀏覽頁面選擇今天要讀的詩篇 23 篇。她點選詩篇書卷，再選擇第 23 章，閱讀完整經文並查看相關主題。

**使用者旅程**:
1. 開啟手機瀏覽器進入網站
2. 點擊底部導航「瀏覽」
3. 在書卷列表中點選「詩篇」
4. 在章節網格中點選「23」
5. 閱讀詩篇 23 篇全文
6. 點擊「下一章」繼續閱讀

---

#### 情境 4：慕道友提問

> 小華聽朋友提到聖經，想知道「耶穌是誰」。他打開 Bible RAG 首頁，直接輸入「耶穌是誰？」系統用淺顯易懂的語言回答，並附上福音書的相關經文。小華點擊其中一段經文，開始閱讀馬太福音的內容。

**使用者旅程**:
1. 進入首頁 (問答頁面)
2. 輸入問題「耶穌是誰？」
3. 等待回應，閱讀 LLM 回答
4. 點擊「馬太福音」相關段落
5. 閱讀經文內容
6. 使用麵包屑導航回到問答頁

| 項目 | 位置 | 功能 |

## 7. 無障礙設計

### 7.1 WCAG 2.1 AA 標準

本專案遵循 WCAG 2.1 AA 等級標準，確保以下原則：

| 原則 | 說明 | 實作重點 |
|------|------|----------|
| **可感知** | 資訊和介面元件可被感知 | 替代文字、字幕、色彩對比 |
| **可操作** | 介面可用多種方式操作 | 鍵盤導航、足夠時間、避免閃爍 |
| **可理解** | 資訊和操作可被理解 | 一致性、錯誤預防、說明文字 |
| **穩健性** | 內容可被多種技術解讀 | 語意化標記、ARIA 標籤 |

### 7.2 鍵盤導航

| `Escape` | 關閉對話框或下拉選單 |

#### 7.2.3 焦點管理

| 情境 | 焦點行為 |
|------|----------|

```css
/* 焦點環樣式 */

  outline: 2px solid var(--primary-500);

}

/* 移除滑鼠點擊的焦點環 */
:focus:not(:focus-visible) {
  outline: none;
}
```
